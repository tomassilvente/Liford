1. Stack tecnológico

Qué tecnología usa la app y para qué sirve cada una.

Next.js 16 — App Router

Framework principal. Cada carpeta dentro de src/app/ es una ruta URL. Los componentes pueden ser de servidor (se ejecutan en el backend, tienen acceso a DB) o de cliente (se ejecutan en el browser, manejan interactividad).

Prisma ORM

Capa de acceso a la base de datos. Permite hacer queries tipadas sin escribir SQL. El schema define todos los modelos. Se ejecuta siempre en el servidor, nunca en el browser.

PostgreSQL

Base de datos relacional. Guarda todo: usuarios, wallets, transacciones, inversiones, clientes, sesiones fotográficas, snapshots de patrimonio, etc.

TypeScript

JavaScript con tipos. Permite que el editor detecte errores antes de ejecutar el código. Prisma genera automáticamente los tipos de todos los modelos.

Tailwind CSS

Estilos via clases en el HTML. En lugar de archivos CSS separados, las clases (bg-neutral-800, text-white, rounded-xl) aplican estilos directamente.

Sonner

Librería de toasts (notificaciones). Cuando guardás algo o hay un error, el mensaje flotante que aparece arriba viene de acá. Requiere el componente <Toaster> en el layout.

Recharts

Librería de gráficos para React. Los gráficos del dashboard de finanzas (barras, líneas, donut) están hechos con Recharts.

googleapis

SDK oficial de Google. Usado para Google Calendar (crear/actualizar/borrar eventos) y Google Drive (crear carpetas, listar y descargar fotos).

@anthropic-ai/sdk

SDK de Anthropic. Usado en dos lugares: el QuickAdd de finanzas (parsea mensajes en lenguaje natural con Claude Haiku) y el Editor de Stories (genera HTML con Claude Sonnet).

jose

Librería para JWT (JSON Web Tokens). Crea y verifica los tokens de sesión de usuario firmados con SESSION_SECRET.

bcryptjs

Hashea contraseñas. Las contraseñas se guardan hasheadas en la DB, nunca en texto plano.

xlsx + adm-zip + better-sqlite3

Usadas en el importador. xlsx lee archivos Excel, adm-zip descomprime .mmbackup, better-sqlite3 lee la DB SQLite interna del archivo de Money Manager.

2. Estructura del proyecto

liford/ (raíz)
  • src/app/(finanzas)/Route Group de finanzas. El paréntesis hace que "(finanzas)" no aparezca en la URL. Tiene su propio layout con sidebar diferente al de fotografía.
  • src/app/(fotografia)/Route Group de fotografía. Mismo concepto. Las URLs son /fotografia/sesiones, /fotografia/clientes, etc.
  • src/app/api/finanzas/Todas las API routes del módulo financiero. Son funciones de servidor que responden peticiones HTTP desde el browser.
  • src/app/api/fotografia/API routes del módulo fotográfico: sesiones, clientes, Drive, editor con Claude.
  • src/app/api/auth/API routes de autenticación: login, logout, registro, cambio de contraseña, /me.
  • src/components/finanzas/Componentes React reutilizables de finanzas (formularios, filas, gráficos, QuickAdd).
  • src/components/fotografia/Componentes React de fotografía (SessionForm, ClientRow, PhotoBrowser, DesignEditor).
  • src/components/ui/Componentes de UI compartidos entre ambos módulos: Sidebar, BottomNav, LogoutButton.
  • src/lib/Funciones de servidor: auth, DB, parsers, Google Calendar, Google Drive, cotizaciones, etc.
  • src/generated/prisma/Tipos TypeScript auto-generados por Prisma. NO se edita manualmente. Se regenera con npx prisma generate.
  • prisma/schema.prismaFuente de verdad de la estructura de la DB. Define todos los modelos, campos, tipos y relaciones.
  • public/Archivos estáticos. logo-stomas.png se usa en el editor de stories como branding en los diseños generados.
  • scripts/google-auth.mjsScript que se corre una única vez para obtener el refresh_token de Google OAuth. No forma parte del servidor.

3. Autenticación

Cómo funciona el login y cómo sabe la app quién sos en cada request.

Flujo completo de autenticación
1
Usuario envía usuario + contraseña
POST /api/auth/login con { username, password }.
↓
2
Verificación de contraseña
Se busca el usuario en la DB. bcryptjs.compare() verifica que el hash guardado coincida con la contraseña ingresada.
↓
3
Creación del JWT
jose crea un token firmado con SESSION_SECRET. El payload contiene { userId, username, displayName }. Expira en 30 días.
↓
4
Cookie httpOnly
El token se guarda en la cookie "liford_session" (httpOnly: true = JavaScript del browser no puede leerla, solo el servidor).
↓
5
Cada request verificada
requireSession() (para páginas) o getApiSession() (para APIs) leen la cookie, verifican el JWT y devuelven el payload. Sin cookie válida → redirect a /login o 401.
src/lib/jwt.ts

encryptToken() crea el JWT. decryptToken() lo verifica. Define la interface SessionPayload { userId, username, displayName }.

src/lib/session.ts

createSession() guarda la cookie. getSession() la lee y devuelve el payload. deleteSession() la borra al hacer logout.

src/lib/auth.ts

requireSession() para Server Components (redirige si no autenticado). getApiSession() para API routes (devuelve null si no autenticado).

src/lib/db.ts

Singleton de Prisma Client. Reutiliza la conexión a DB en desarrollo para no saturar el pool de conexiones con los hot-reloads.

4. Base de datos completa

Modelos de Finanzas

User

User — Un usuario de la app. Todo está asociado a un userId.
idStringCUID único.PK
usernameStringNombre de usuario para login. Único en la DB.
displayNameString?Nombre para mostrar en el sidebar.opcional
passwordHashStringContraseña hasheada con bcrypt. Nunca texto plano.

Wallet (Billetera)

Wallet — Billetera en ARS o USD. Su balance se actualiza con cada transacción.
idStringCUID.PK
userIdStringPropietario.FK → User
nameStringNombre descriptivo (ej: "Mercado Pago", "Santander").
currencyCurrencyARS o USD.
balanceFloatSaldo actual. Se incrementa/decrementa atómicamente junto con cada transacción.

ForeignAccount (Cuenta Foránea)

ForeignAccount — Cuentas en el exterior: Wise, Payoneer, broker, etc.
id / userId / nameStringIgual que Wallet.
currencyStringCualquier moneda (no limitado al enum Currency).
balanceFloatSaldo en la moneda de la cuenta.
notesString?Notas adicionales.opcional

Transaction (Transacción)

Transaction — Central de finanzas. Registra gastos, ingresos, compras de activos Y cobros de fotografía.
idStringCUID.PK
userIdStringPropietario.FK → User
typeTransactionTypeEXPENSE, INCOME, STOCK_PURCHASE, CRYPTO_PURCHASE.
sourceTransactionSourcePERSONAL (finanzas personales) o PHOTOGRAPHY (cobro de sesión fotográfica).
amountFloatMonto positivo.
currencyCurrencyARS o USD.
categoryStringCategoría libre (ej: "Alimentación", "Sueldo", "Fotografía").
descriptionString?Descripción del gasto o ingreso.opcional
dateDateTimeFecha de la transacción.
sessionIdString?Si source=PHOTOGRAPHY, vincula al Session que originó el ingreso.opcionalFK → Session

Investment (Inversión)

Investment — Posición en acciones o cripto. El precio actual se consulta en Yahoo Finance en tiempo real.
userIdStringPropietario.FK → User
typeInvestmentTypeSTOCK (acción) o CRYPTO (criptomoneda).
tickerStringSímbolo del activo. Ej: "GGAL", "BTC-USD", "AAPL".
quantityFloatCantidad de unidades en cartera.
avgBuyPriceFloatPrecio promedio de compra en USD. Se recalcula con cada compra adicional.

RecurringExpense (Gasto recurrente)

RecurringExpense — Gastos o ingresos que se aplican automáticamente cada mes.
description / amount / currency / category—Igual que una transacción.
dayOfMonthIntDía del mes en que se aplica (ej: 5 = el día 5 de cada mes).
isActiveBooleanSi está activo. Los inactivos no se aplican.
lastAppliedDateTime?Cuándo fue la última vez que se aplicó. Evita duplicados.opcional
walletId / foreignAccountIdString?Cuenta a debitar/acreditar al aplicar.opcional

Budget (Presupuesto) · SavingsGoal (Meta) · WealthSnapshot

Budget

Presupuesto mensual por categoría. Tiene category, limit (monto máximo) y month (YYYY-MM). La página de presupuesto compara los gastos reales del mes contra cada límite.

SavingsGoal

Metas de ahorro. Tiene name, targetAmount, currentAmount y currency. Se actualiza manualmente al ir aportando.

WealthSnapshot

Snapshot mensual del patrimonio. Se crea automáticamente una vez por mes al abrir finanzas. Guarda walletsARS, foreignUSD, portfolioUSD, totalARS, totalUSD. Alimenta el gráfico de evolución de patrimonio.

Modelos de Fotografía

Client

Client — Cliente del fotógrafo (equipo, persona, empresa).
id / userIdStringIdentificador y propietario.PK FK → User
nameStringNombre del cliente o equipo.
instagramString?Usuario de Instagram (sin @).opcional
phoneString?Teléfono de contacto.opcional
notesString?Notas libres sobre el cliente.opcional
sessionsSession[]Todas las sesiones de este cliente.

Session

Session — Sesión fotográfica. Modelo central del módulo.
idStringCUID.PK
clientIdStringCliente al que pertenece.FK → Client
typeSessionTypeSPORT, EVENT u OTHER.
eventNameString?Nombre del evento (solo para tipo EVENT).opcional
dateDateTimeFecha y hora. Guardada en UTC. El frontend maneja el timezone Argentina (-03:00).
durationMinutesInt?Duración. Usado para calcular el fin del evento en Calendar.opcional
priceFloatPrecio pactado con el cliente.
currencyCurrencyARS o USD.
statusSessionStatusEstado actual en el workflow. Default: PENDING.
photosDeliveredInt?Cantidad de fotos entregadas al finalizar.opcional
googleCalendarIdString?ID del evento en Google Calendar para actualizarlo o borrarlo.opcional
driveUrlString?URL de la carpeta en Google Drive con las fotos entregadas.opcional
notesString?Notas internas sobre la sesión.opcional
transactionsTransaction[]Ingresos de finanzas vinculados (se crea uno al marcar PAID).

Estados de una sesión

PENDING→ CONFIRMED→ SHOT→ DELIVERED→ PAID→ COMPLETED
Al pasar a PAID: se crea automáticamente una Transaction con source=PHOTOGRAPHY en Finanzas, vinculada a la sesión via sessionId. Aparece en /finanzas/ingresos con badge azul "Fotografía".

Conexión entre módulos

Cómo se conectan finanzas y fotografía
A
Sesión marcada como PAID en Fotografía
El PATCH de sessions/[id] detecta el cambio de estado.
↓
B
Se crea una Transaction en Finanzas
db.transaction.create() con type=INCOME, source=PHOTOGRAPHY, amount=session.price, currency=session.currency, category="Fotografía", sessionId=session.id.
↓
C
El ingreso aparece en /finanzas/ingresos
La query de ingresos incluye source IN [PERSONAL, PHOTOGRAPHY]. Los de fotografía muestran badge azul y no tienen botón de editar/borrar (solo se gestionan desde fotografía).
↓
D
El dashboard de finanzas lo cuenta
Los ingresos PHOTOGRAPHY suman al total mensual de ingresos y aparecen en el gráfico de Ingresos vs Gastos.

5. Módulo Finanzas

Layout

src/app/(finanzas)/layout.tsxServer Component async

Envoltorio de todas las páginas de finanzas. Al cargarse, ejecuta en paralelo 4 cosas:

  • requireSession()Verifica auth. Si no hay sesión → redirect /login.
  • applyRecurringExpenses(userId)Aplica los gastos recurrentes del mes que aún no fueron aplicados. Se ejecuta silenciosamente en cada carga.
  • saveWealthSnapshotIfNeeded(userId)Guarda el snapshot mensual de patrimonio si aún no existe para el mes actual.
  • db.wallet.findMany() + db.foreignAccount.findMany()Carga las billeteras y cuentas para pasarlas al componente QuickAdd.

Renderiza: Sidebar (oculto en mobile), main, BottomNav (visible en mobile), QuickAdd (botón flotante +), Toaster.

Dashboard — /finanzas

src/app/(finanzas)/finanzas/page.tsxServer Component

La página más completa de la app. Hace 6 queries en paralelo con Promise.all() y calcula todo el estado financiero.

  • Patrimonio totalSuma billeteras ARS, cuentas foráneas USD, y portfolio (precio actual × cantidad de cada activo desde Yahoo Finance).
  • Variación diaria del portfolioSuma el changeAmount de cada activo × cantidad para mostrar cuánto cambió el portfolio en el día.
  • Balance del mesFiltra transacciones del mes actual. Calcula ingresos, gastos, balance (ingresos - gastos) y tasa de ahorro ((ingresos - gastos) / ingresos × 100).
  • Comparación vs mes anteriorCalcula el % de variación de ingresos y gastos respecto al mes anterior.
  • Gastos por categoríaAgrupa gastos del mes por categoría, toma las 8 más grandes. Alimenta el CategoryChart (donut).
  • Gráfico mensualAgrupa ingresos y gastos de los últimos 6 meses. Alimenta el MonthlyChart (barras).
  • Evolución del patrimonioLee los últimos 12 WealthSnapshots. Alimenta el WealthChart (líneas).
  • Últimas 8 transaccionesLas más recientes para el widget de historial rápido.

Gastos e Ingresos

/finanzas/gastos

Formulario (GastoForm) + filtro de mes (MonthFilter) + lista de transacciones (TransactionRow). Filtra por mes via searchParam ?mes=YYYY-MM. Muestra el total por moneda. TransactionRow permite editar descripción, categoría, monto inline y eliminar con confirmación de dos pasos.

/finanzas/ingresos

Idéntico a gastos pero para ingresos. Query incluye source IN [PERSONAL, PHOTOGRAPHY] para mostrar también los cobros de sesiones fotográficas. Los de PHOTOGRAPHY tienen badge azul y son de solo lectura (sin editar/borrar).

Billeteras y Cuentas Foráneas

src/app/(finanzas)/finanzas/billeteras/page.tsxServer Component

Muestra WalletCard y ForeignAccountCard para cada cuenta. Las wallets muestran nombre, moneda y balance. Las cuentas foráneas permiten editar el balance manualmente (ya que son cuentas externas que no se actualizan automáticamente).

  • WalletCard / WalletFormCard de billetera con botón para editar nombre y eliminar. WalletForm crea nuevas billeteras. El balance de las wallets se actualiza automáticamente con cada transacción que las referencia.
  • ForeignAccountCard / ForeignAccountFormSimilar pero para cuentas foráneas. Tiene un campo extra para actualizar el balance manualmente (útil para sincronizar con Wise, Payoneer, etc.).

Inversiones

src/app/(finanzas)/finanzas/inversiones/page.tsxServer Component
  • fetchCotizaciones(tickers)Al cargar la página, llama a Yahoo Finance para obtener el precio actual de cada ticker en cartera. Cache de 5 minutos (next: { revalidate: 300 }).
  • PnL totalCalcula el resultado global: (precio actual × cantidad) - (precio promedio compra × cantidad) para cada activo. Muestra total invertido, valor actual y ganancia/pérdida en $ y %.
  • InversionCardCard por activo mostrando ticker, tipo (stock/cripto), cantidad, precio promedio, precio actual, variación del día. Tiene gráfico de historial de precios (PriceHistoryChart, llama a /api/finanzas/cotizaciones/[ticker]).
  • PortfolioAllocationChartGráfico de dona que muestra la distribución del portfolio por activo.
  • InversionFormPara agregar un activo. Si ya existe el ticker en cartera, recalcula el precio promedio ponderado.

Presupuesto

src/app/(finanzas)/finanzas/presupuesto/Server + Client

Permite definir un límite mensual por categoría y ver cuánto se lleva gastado. BudgetManager (componente cliente) muestra una barra de progreso por categoría: verde si está bajo el límite, amarillo cerca, rojo si lo superó. Permite crear, editar y eliminar límites.

Metas de ahorro

src/app/(finanzas)/finanzas/metas/Server + Client

GoalsManager (componente cliente) gestiona metas de ahorro. Cada meta tiene nombre, monto objetivo, monto actual y moneda. Se puede aportar dinero manualmente incrementando el currentAmount. Muestra % de progreso.

Gastos Recurrentes

src/app/(finanzas)/finanzas/recurrentes/Server + Client

RecurringManager lista y gestiona los gastos e ingresos recurrentes. Cada uno tiene: descripción, monto, moneda, categoría, día del mes, cuenta a debitar/acreditar, y si está activo. La lógica de aplicación está en apply-recurring.ts y se ejecuta automáticamente al abrir finanzas.

Importar / Exportar

src/app/(finanzas)/finanzas/importar/Server + Client
  • ImportFormAcepta 3 formatos de archivo: Excel (.xlsx), JSON, y .mmbackup (exportación de la app Money Manager). El archivo se sube a POST /api/finanzas/import que lo parsea y devuelve una preview de las transacciones detectadas.
  • Preview + ConfirmaciónEl usuario ve las transacciones que se van a importar antes de confirmar. POST /api/finanzas/import/confirm las inserta en la DB en una Prisma transaction atómica.
  • ExportButtonDescarga todas las transacciones en formato Excel via GET /api/finanzas/export. Usa la librería xlsx para generar el archivo.
  • import-parser.tsLógica de parseo de los 3 formatos (ver sección 6).

QuickAdd con IA

src/components/finanzas/QuickAdd.tsxClient Component

El botón "+" flotante en la esquina inferior derecha. La funcionalidad más interactiva del módulo.

  • Modo manualFormulario con campos: tipo (gasto/ingreso), monto, cuenta, categoría, descripción. Llama a /api/finanzas/gastos o /api/finanzas/ingresos según el tipo.
  • Modo IA (texto libre)El usuario escribe en lenguaje natural: "gasté 5000 en el super". Se llama a /api/finanzas/transactions con el texto. El servidor usa Claude Haiku para parsear el mensaje y extraer: tipo, monto, moneda, descripción y categoría. El resultado pre-completa el formulario para que el usuario confirme.
  • Atajos de tecladoTecla Escape cierra el QuickAdd. El campo de monto recibe foco automáticamente al abrir.
Claude Haiku: se usa claude-haiku-4-5-20251001 para el parseo de transacciones porque es más rápido y económico que Sonnet para esta tarea simple y estructurada.

6. Librerías de servidor

Funciones reutilizables que viven en src/lib/ y pueden ser importadas desde cualquier Server Component o API route.

cotizaciones.ts — Precios de activos

src/lib/cotizaciones.tsServer Library
  • fetchCotizaciones(tickers: string[]): Promise<Record<string, Cotizacion>>Llama a Yahoo Finance en paralelo para cada ticker. Devuelve un mapa ticker → datos. Cache de 5 minutos con next: { revalidate: 300 }. Si un ticker falla, igual devuelve el resto con el campo error.Datos devueltos: precio actual, variación del día ($, %), rango del día (high/low), cierre anterior, rango de 52 semanas, estado del mercado (REGULAR, PRE, POST, CLOSED).

apply-recurring.ts — Gastos automáticos

src/lib/apply-recurring.tsServer Library
  • applyRecurringExpenses(userId: string)Busca RecurringExpenses que: están activos, su dayOfMonth es menor o igual al día de hoy, y no fueron aplicados este mes (lastApplied < inicio del mes actual). Los aplica todos en una sola transacción de Prisma ($transaction): crea las transacciones, actualiza los saldos de las wallets/cuentas, y actualiza lastApplied. Se llama en el layout de finanzas en cada visita.
Idempotente: si se llama varias veces el mismo mes, el filtro lastApplied < thisMonthStart evita duplicados.

save-wealth-snapshot.ts — Snapshot de patrimonio

src/lib/save-wealth-snapshot.tsServer Library
  • saveWealthSnapshotIfNeeded(userId: string)Verifica si ya existe un WealthSnapshot para el mes actual. Si no existe, calcula el patrimonio completo (billeteras ARS, cuentas foráneas USD, portfolio al precio actual de Yahoo Finance) y lo guarda. Esto alimenta el gráfico de evolución de patrimonio en el dashboard.Solo crea un snapshot por mes. Si el usuario abre la app el día 1 y el 15, solo hay un snapshot del mes (el del día 1).

import-parser.ts — Importador de datos

src/lib/import-parser.tsServer Library

Parsea 3 formatos de archivo diferentes y los convierte al mismo formato interno ImportedTransaction[].

  • parseExcel(buffer)Lee un .xlsx con SheetJS. Detecta columnas automáticamente por nombre (acepta: "fecha", "date", "monto", "amount", "tipo", "type", "categoría", etc.). Maneja fechas en múltiples formatos (DD/MM/YYYY, YYYY-MM-DD, objetos Date).
  • parseJSON(text)Parsea un JSON. Acepta array directo o { transactions: [...] } o { data: [...] }. Reutiliza la misma lógica de detección de columnas que Excel.
  • parseMmbackup(buffer)El más complejo. Un .mmbackup es un ZIP con 8 bytes de prefijo propietario. Lo descomprime, extrae MyFinance.db (SQLite), lee las tablas 'transaction', 'category' y 'sync_link', resuelve el nombre de categoría de cada transacción y convierte los montos de centavos a pesos. Maneja cuentas ARS y USD. Ignora transacciones de cripto nativas de la app.

expense-parser.ts — Parseo con IA

src/lib/expense-parser.tsServer Library
  • parseTransaction(text: string): Promise<ParsedTransaction | null>Llama a Claude Haiku con el texto libre del usuario. El prompt le pide que devuelva JSON con: type, amount, currency, description, category. Detecta "k" para multiplicar por 1000, interpreta "verdes" / "cables" como USD, infiere si es gasto o ingreso por el verbo. Extrae el JSON incluso si Claude lo envuelve en bloques de código markdown.
Ejemplo: "me pagaron 50k de fotos al Farsa" → { type: "INCOME", amount: 50000, currency: "ARS", description: "Fotos al Farsa", category: "Fotografía" }

google-calendar.ts

src/lib/google-calendar.tsServer Library
  • createCalendarEvent({ summary, description, startISO, durationMinutes })Crea un evento. El timezone es siempre America/Argentina/Buenos_Aires. Devuelve el ID del evento para guardarlo en Session.googleCalendarId. Si las credenciales no están configuradas, devuelve null sin romper.
  • updateCalendarEvent(eventId, { startISO, durationMinutes })Actualiza fecha/hora de un evento. Llamado cuando se edita la fecha de una sesión.
  • deleteCalendarEvent(eventId)Elimina el evento. Llamado cuando se borra una sesión.
Tolerancia a fallos: todas las funciones tienen try/catch. Si falla Google, la operación en DB sigue adelante. Los errores se loguean en consola del servidor.

google-drive.ts

src/lib/google-drive.tsServer Library
  • findOrCreateFolder(name, parentId?)Función interna. Busca si existe una carpeta con ese nombre en el padre. Si existe, devuelve su ID. Si no, la crea. Evita duplicados cuando se fotografía al mismo equipo en distintas fechas.
  • createSessionFolder({ clientName, date })Crea la estructura: [Nombre Equipo] / [DD-MM-YYYY]. Nivel 1: carpeta del equipo (reutilizada si existe). Nivel 2: subcarpeta con la fecha. Devuelve { id, url } de la carpeta de fecha.
  • listPhotosInFolder(folderId)Lista solo imágenes (mimeType contains 'image/') en una carpeta. Hasta 200 fotos, ordenadas por nombre.
  • listDriveFolder(folderId)Lista carpetas E imágenes. "root" muestra el Drive raíz (Mi unidad). Carpetas primero, luego imágenes. Usado por el browser de Drive en el editor.
  • getThumbnailStream(fileId)Obtiene el thumbnail de Drive. Los thumbnails de Drive requieren autenticación, por lo que esta función los descarga server-side con el access token y devuelve el stream para proxearlo. También aumenta el tamaño a s600 para mejor calidad.
  • extractFolderId(driveUrl)Extrae el ID de una URL de Drive: /folders/([a-zA-Z0-9_-]+)

8. Flujos de datos clave

Registrar un gasto via QuickAdd con IA

1
Usuario escribe: "gasté 5000 en el super"
En el QuickAdd (botón + flotante), modo texto libre.
↓
2
POST /api/finanzas/transactions con el texto
expense-parser.ts llama a Claude Haiku. El prompt le pide JSON con type, amount, currency, description, category. Claude responde: { type: "EXPENSE", amount: 5000, currency: "ARS", description: "Supermercado", category: "Alimentación" }.
↓
3
El formulario se pre-completa
QuickAdd.tsx recibe el resultado y llena los campos. El usuario puede revisar y corregir antes de confirmar.
↓
4
POST /api/finanzas/gastos
Crea la Transaction en DB y decrementa el balance de la wallet seleccionada en una sola $transaction atómica de Prisma.

Crear una sesión fotográfica

1
SessionForm.tsx → POST /api/fotografia/sessions
Fecha + hora combinadas con offset -03:00. Verifica que el cliente exista y pertenezca al usuario.
↓
2
Se crea la sesión en la DB
db.session.create() con status PENDING.
↓
3
Promise.all(): Drive + Calendar en paralelo
createSessionFolder() crea [Equipo]/[DD-MM-YYYY] en Drive. createCalendarEvent() crea el evento en Google Calendar. Si alguno falla, el otro sigue.
↓
4
Guarda IDs externos + actualiza UI
Si Drive devolvió URL → Session.driveUrl. Si Calendar devolvió ID → Session.googleCalendarId. router.refresh() actualiza la lista.

Marcar sesión como PAID → ingreso automático en Finanzas

1
SessionRow edición → status: "PAID"
PATCH /api/fotografia/sessions/[id] con { status: "PAID", ...otros campos }.
↓
2
Detección del cambio de estado
body.status === "PAID" && existing.status !== "PAID" && existing._count.transactions === 0 → condición para crear el ingreso.
↓
3
db.transaction.create() en Finanzas
type=INCOME, source=PHOTOGRAPHY, amount=session.price, currency, category="Fotografía", sessionId=id.
↓
4
Aparece en /finanzas/ingresos con badge "Fotografía"
La query de ingresos incluye source IN [PERSONAL, PHOTOGRAPHY]. Las transacciones de fotografía son de solo lectura.

10. Setup inicial

1
Clonar e instalar
npm install en la raíz del proyecto.
2
Configurar .env
Copiar .env.example a .env y completar DATABASE_URL y SESSION_SECRET como mínimo.
3
Crear la DB
npx prisma migrate dev crea todas las tablas. npx prisma generate genera los tipos TypeScript.
4
Configurar Google (opcional)
Completar GOOGLE_CLIENT_ID y GOOGLE_CLIENT_SECRET. Correr node scripts/google-auth.mjs para obtener el REFRESH_TOKEN con scopes de Calendar + Drive.
5
Correr
npm run dev → localhost:3000. Registrar usuario en /api/auth/register (o usar el formulario si está implementado).
Sin Google: la app funciona igual. Las sesiones se crean sin carpeta de Drive ni evento de Calendar. Sin ANTHROPIC_API_KEY: el QuickAdd de IA y el Editor de Stories no funcionan.
